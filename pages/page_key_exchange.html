Key Exchange

Overview

The Key Exchange Problem

Symmetric encryption is fast and secure, but requires both parties to share the exact same key. If they have never met, how do they establish this key over the public internet without an eavesdropper stealing it? Sending the key in plain text compromises the entire system.

Key Exchange algorithms solve this problem. They allow two parties (Alice and Bob) to create a shared secret key over an insecure connection, even when an eavesdropper (Eve) is watching every packet of data they transmit.

Analogy

The Paint-Mixing Metaphor

The standard way to understand key exchange (specifically the Diffie-Hellman algorithm) is by using paint colors:

  1. Alice and Bob agree on a common Public Color (e.g., Yellow).
  2. Alice and Bob each select a secret Private Color (Alice chooses Red, Bob chooses Blue).
  3. Alice mixes her private color (Red) with the public color (Yellow) to get Orange, and sends it to Bob. Bob mixes his private color (Blue) with the public color (Yellow) to get Light Blue, and sends it to Alice.
  4. Eve the eavesdropper sees the public Yellow and the mixtures (Orange and Light Blue). However, unmixing colors is mathematically hard.
  5. Finally, Alice adds her private color (Red) to Bob's mixture (Light Blue), and Bob adds his private color (Blue) to Alice's mixture (Orange). Both arrive at the exact same secret color (Purple)!

Protocols

Key Exchange Algorithms

Diffie-Hellman (DH)

The original protocol invented in 1976. It uses the mathematics of discrete logarithms to allow two parties to securely generate a secret key.

ECDH (Elliptic Curve Diffie-Hellman)

The modern standard. Instead of large integers, it uses elliptic curves. ECDH provides equivalent security to classical DH but with much smaller keys, leading to faster handshakes and lower CPU usage (essential for mobile devices).
