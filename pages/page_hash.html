Hash Functions

Overview

What is a Cryptographic Hash?

A cryptographic hash function is a mathematical process that takes an input of any size (such as a single word or an entire file) and converts it into a fixed-size string of characters (the hash output).

Think of it like a digital fingerprint. Just as a physical fingerprint uniquely identifies a human without revealing their personality or thoughts, a cryptographic hash uniquely identifies a piece of data without exposing its content. Importantly, hashing is a one-way street—you can easily generate a fingerprint from a person, but you cannot rebuild the person from the fingerprint.

Properties

Core Requirements of Cryptographic Hashes

One-Way: It must be mathematically impossible to reverse the hash. You cannot reconstruct "Input A" from "Hash A".
Deterministic: The exact same input will always yield the exact same hash output.
Collision Resistant: It must be practically impossible to find two different inputs that produce the identical output hash.
Avalanche Effect: Even a tiny modification to the input data must cause a massive, unpredictable change in the resulting hash.
Standards

Common Hashing Algorithms

SHA-2 (Secure Hash Algorithm 2)

The global cryptographic standard. It includes SHA-256 and SHA-512. It is widely used in secure web connections (HTTPS), software signing, and blockchains (like Bitcoin).

SHA-3 (Secure Hash Algorithm 3)

The newest standard family. It is mathematically distinct from SHA-2, designed to provide a secure alternative in case SHA-2 is ever compromised in the future.

MD5 & SHA-1 (Legacy / Insecure)

Old algorithms that are now considered cryptographically broken. Because computers can easily find "collisions" (two different files generating the same hash) in minutes, they are strictly used for fast non-security checksums and are never used for authentication or passwords.
